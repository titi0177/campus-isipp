INSTRUCCIONES PARA CORREGIR EL ERROR 403 EN SUPABASE

PROBLEMA:
El profesor ve error 403 (Forbidden) al intentar cargar calificaciones.
Esto sucede porque la tabla enrollment_grades tiene RLS habilitado pero las políticas
no permiten al profesor leer/escribir sus calificaciones.

SOLUCIÓN PASO A PASO:

1. ACCEDER A SUPABASE SQL EDITOR
   - Va a: https://app.supabase.com
   - Selecciona tu proyecto
   - Ve a "SQL Editor" en el menu izquierdo
   - Click en "New Query"

2. COPIAR Y EJECUTAR LAS MIGRACIONES

   PASO 2A: Ejecuta esta consulta (Verificación)
   ==============================================================
   -- Ver estructura actual de enrollment_grades
   SELECT column_name, data_type, is_nullable 
   FROM information_schema.columns 
   WHERE table_name = 'enrollment_grades' 
   ORDER BY ordinal_position;
   
   -- Ver políticas RLS actuales
   SELECT schemaname, tablename, policyname, permissive, roles, qual, with_check
   FROM pg_policies 
   WHERE tablename = 'enrollment_grades' 
   ORDER BY policyname;
   ==============================================================
   Verifica que existan las columnas: grade_1 a grade_6, partial_grade, etc.

   PASO 2B: Si la tabla NO existe, corre esto primero
   ==============================================================
   CREATE TABLE IF NOT EXISTS public.enrollment_grades (
     id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
     enrollment_id UUID NOT NULL REFERENCES public.enrollments(id) ON DELETE CASCADE,
     
     grade_1 NUMERIC(4,2),
     grade_2 NUMERIC(4,2),
     grade_3 NUMERIC(4,2),
     grade_4 NUMERIC(4,2),
     grade_5 NUMERIC(4,2),
     grade_6 NUMERIC(4,2),
     
     partial_grade NUMERIC(4,2),
     partial_status TEXT CHECK (partial_status IN ('regular', 'promocionado', 'desaprobado')),
     
     final_grade NUMERIC(4,2),
     final_status TEXT CHECK (final_status IN ('regular', 'promocionado', 'desaprobado')),
     
     attempt_number INTEGER NOT NULL DEFAULT 1,
     created_at TIMESTAMPTZ NOT NULL DEFAULT NOW(),
     updated_at TIMESTAMPTZ NOT NULL DEFAULT NOW(),
     
     UNIQUE(enrollment_id)
   );
   
   CREATE INDEX IF NOT EXISTS idx_enrollment_grades_enrollment ON public.enrollment_grades(enrollment_id);
   
   ALTER TABLE public.enrollment_grades ENABLE ROW LEVEL SECURITY;
   ==============================================================

   PASO 2C: CORREGIR RLS - Eliminar políticas antiguas y crear nuevas
   ==============================================================
   -- Eliminar todas las políticas anteriores
   DROP POLICY IF EXISTS "enrollment_grades_student_read" ON public.enrollment_grades;
   DROP POLICY IF EXISTS "enrollment_grades_professor_read" ON public.enrollment_grades;
   DROP POLICY IF EXISTS "enrollment_grades_admin_all" ON public.enrollment_grades;
   DROP POLICY IF EXISTS "enrollment_grades_professor_write" ON public.enrollment_grades;
   DROP POLICY IF EXISTS "enrollment_grades_professor_update" ON public.enrollment_grades;
   DROP POLICY IF EXISTS "enrollment_grades_professor_insert" ON public.enrollment_grades;
   DROP POLICY IF EXISTS "enrollment_grades_professor_insert_strict" ON public.enrollment_grades;
   DROP POLICY IF EXISTS "enrollment_grades_professor_read_strict" ON public.enrollment_grades;
   DROP POLICY IF EXISTS "enrollment_grades_professor_update_strict" ON public.enrollment_grades;

   -- Crear políticas PERMISIVAS (temporales, hasta que user_id esté poblado)
   CREATE POLICY "enrollment_grades_allow_select" ON public.enrollment_grades
     FOR SELECT USING (true);

   CREATE POLICY "enrollment_grades_allow_insert" ON public.enrollment_grades
     FOR INSERT WITH CHECK (true);

   CREATE POLICY "enrollment_grades_allow_update" ON public.enrollment_grades
     FOR UPDATE USING (true) WITH CHECK (true);

   -- Admin acceso total
   CREATE POLICY "enrollment_grades_admin_all" ON public.enrollment_grades
     FOR ALL USING (public.is_admin());
   ==============================================================

   PASO 2D: Agregar columnas faltantes a enrollments
   ==============================================================
   ALTER TABLE public.enrollments ADD COLUMN IF NOT EXISTS academic_year INTEGER;
   ALTER TABLE public.enrollments ADD COLUMN IF NOT EXISTS is_recursive BOOLEAN NOT NULL DEFAULT false;
   ALTER TABLE public.enrollments ADD COLUMN IF NOT EXISTS status TEXT NOT NULL DEFAULT 'active' CHECK (status IN ('active', 'inactive', 'completed', 'dropped'));
   ALTER TABLE public.enrollments ADD COLUMN IF NOT EXISTS attempt_number INTEGER NOT NULL DEFAULT 1;
   ALTER TABLE public.enrollments ADD COLUMN IF NOT EXISTS division TEXT;
   
   CREATE INDEX IF NOT EXISTS idx_enrollments_academic_year ON public.enrollments(academic_year);
   ==============================================================

   PASO 2E: Agregar user_id a professors
   ==============================================================
   ALTER TABLE public.professors ADD COLUMN IF NOT EXISTS user_id UUID REFERENCES public.users(id) ON DELETE SET NULL;
   CREATE INDEX IF NOT EXISTS idx_professors_user ON public.professors(user_id);
   ==============================================================

3. VERIFICAR QUE FUNCIONÓ
   - Recarga la página del navegador (Ctrl+F5)
   - Vuelve a intentar cargar calificaciones
   - El error debería desaparecer

4. SI SIGUE DANDO ERROR:
   - Abre Consola del Navegador (F12 → Console)
   - Copia el mensaje de error completo
   - Envía a soporte con el mensaje exacto

NOTA IMPORTANTE:
Las políticas actuales son PERMISIVAS para que funcione ahora.
Una vez que tengas professors.user_id poblado correctamente,
deberías cambiar a políticas más restrictivas (ver archivo CAMBIOS_REALIZADOS.txt)
