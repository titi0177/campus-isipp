RESTRICCIONES EN ENROLLMENTS - DECISIÓN FINAL

SITUACION ANTERIOR:
- Sacamos TODOS los constraints para permitir reinscripciones
- Causó error "duplicate key" al intentar reinscribirse

PROBLEMA CON "SIN CONSTRAINTS":
Sin constraintsal nivel de BD, cualquier bug de código podría permitir:
- 2 inscripciones idénticas (mismo alumno, materia, año, intento)
- Datos inconsistentes
- Problemas de integridad

SOLUCIÓN CORRECTA:
Restaurar UN constraint específico que:
✓ Previene duplicados a nivel de BD
✓ Permite múltiples intentos (1, 2, 3)
✓ Funciona con la lógica de reinscripción

CONSTRAINT CORRECTO:
```sql
ALTER TABLE public.enrollments
ADD CONSTRAINT enrollments_unique_attempt 
UNIQUE (student_id, subject_id, academic_year, attempt_number);
```

QUE PERMITE:
- ✓ (student=1, subject=MATH, year=2024, attempt=1) 
- ✓ (student=1, subject=MATH, year=2024, attempt=2)  ← Recursante
- ✓ (student=1, subject=MATH, year=2025, attempt=2)  ← Mismo año diferente
- ✓ (student=1, subject=MATH, year=2025, attempt=3)  ← Tercer intento

QUE PREVIENE:
- ✗ (student=1, subject=MATH, year=2024, attempt=1) × 2  ← Duplicate
- ✗ (student=1, subject=MATH, year=2024, attempt=2) × 2  ← Duplicate
